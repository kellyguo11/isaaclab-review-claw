## RSL RL Training Re-run (17:35–ongoing)

### Root Cause Analysis
- **Problem**: 45/50 RSL RL tasks failed instantly (~4s) with "Isaac Sim is not installed"
- **Root cause**: `isaacsim` pip package was NOT installed in `env_isaaclab`
  - `sim_launcher.py` checks `importlib.util.find_spec("omni.kit")` before launching Kit
  - Without isaacsim bootstrapping Kit paths, omni.kit not on PYTHONPATH
  - All PhysX-based tasks need Kit (even Newton-configured envs resolve to PhysX via Hydra presets)
  - The 5 successful tasks (Ant, Cartpole, Shadow Vision) ran in an earlier batch when Kit happened to be available
- **Key insight**: Even though env configs have Newton physics, Hydra preset resolution selects PhysX at runtime → needs_kit=True for ALL tasks

### Fixes Applied
1. Installed `isaacsim` 6.0.0.0 pip package
2. Symlinked Kit kernel from packman cache to pip isaacsim
3. Symlinked IsaacSim build extensions (`/home/horde/IsaacSim/_build/linux-x86_64/release/exts`)
4. Created `omni.usdex.libs` 2.2.2 from cached 2.2.1 (NVIDIA CDN returning ACCESS_DENIED)
5. Added ext search paths to all `.kit` experience files pointing to local IsaacSim build extensions
6. Patched `get_isaac_sim_version()` in `version.py` with try/except fallback (pip isaacsim lacks `isaacsim.core.version` extension)
7. Accepted EULA for isaacsim pip package
