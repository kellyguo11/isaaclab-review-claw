## 🐛 BUG FOUND: Headless video rendering produces all-black frames
**Root cause:** After the visualizer refactor (PR #4948), `SimulationContext.render()` no longer calls
`omni.kit.app.get_app().update()`. It only updates visualizers. For headless video recording
(offscreen_render=True), the RTX renderer pipeline is never pumped → replicator annotator gets empty data.

**Fix:** Added `app.update()` call in `SimulationContext.render()` when `self._has_offscreen_render` is True.
File: `source/isaaclab/isaaclab/sim/simulation_context.py`

**Verification:** Before fix: 0/30 non-black frames. After fix: 29/30 non-black frames (only first frame is warmup black).

**Impact:** Affects ALL headless video recording in Isaac Lab (play.py --video --headless), not just our campaign.
This should be submitted as a PR fix.

**Additional notes:**
- Must use Xvfb for Vulkan display (DISPLAY=:2 since :99 was stale)
- L40 GPU IOMMU warning is benign but noted
- `app.update()` is gated on `_has_offscreen_render` so normal headless training (no cameras) is unaffected
