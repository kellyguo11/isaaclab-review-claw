## PR Review Bot Inline Comments Fix
- **Root cause**: Inline comments had `line: null` because:
  1. Bot wasn't specifying `side: "RIGHT"` in review payload
  2. GitHub API requires `line` to match a line in the NEW version (right side of diff)
- **Fix applied** to `webhook/multi-agent.js` and `webhook/server.js`:
  - Added `side: "RIGHT"` requirement for inline comments
  - Clarified `line` must match a line in PR diff
  - Added example JSON payload showing correct structure
  - Instructions to put findings in review body if line not in diff
- Pushed fix to repo

## Technical Notes
- GitHub review API inline comment format:
  ```json
  {
    "path": "src/file.py",
    "line": 42,
    "side": "RIGHT",
    "body": "..."
  }
  ```
- `line` = line number in NEW version of file (not diff position, not old file)
- Line MUST appear in the diff (added or modified lines only)
